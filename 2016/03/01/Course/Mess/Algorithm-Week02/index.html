演算法

透過輸入和演算法也就是計算過程，最後產出輸出，若對於每個 Instance 都能正確輸出後停止，則該演算法是 正確的 ，也就是解決了問題! 在某些情況還有錯誤率的問題，演算法就算不正確也還是有用的!

Instance:

In general, an instance of a problem consists of the input (satisfying whatever constraints are imposed in the problem statement) needed to compute a solution to the problem. - 《Introduction to Algorithms》

可以當作就是一連串的輸入。

算法分析

時間複雜度:

  • $T(n)$ : 所有情況時間複雜度
  • $W(n)$ : 最差情況時間複雜度
  • $A(n)$ : 平均情況時間複雜度
  • $B(n)$ : 最佳情況時間複雜度

後三者只要 $T(n)$ 存在，就會等於 $T(n)$。

Order(量級):
用來表示時間複雜度，Asymptotic notation(漸近符號)

  • 大O符號 $\mathcal O$
    漸近上限，也就是給定一個複雜度函數 $g(n)$ 存在正實數 $c$ 和 $n_0$ 使得所有 $n \ge n_0$ 時， $f(n) \le cg(n)$，則 $f(n) \in O(g(n))$ 也可寫作 $f(n) = O(g(n))$ ，$O(g(n))$ 是由一些複雜度函數 $f(n)$ 所構成的集合。

    $O(g(n)) = \{f(n): $ there exist positive constants $c$ and $n_0$ such that $ 0 \le f(n) \le cg(n) $ for all $ n \ge n_0 \}$

(以下僅寫定義)

  • Omega 符號 $\Omega$
    漸近下限
    $\Omega(g(n)) = \{f(n): $ there exist positive constants $c$ and $n_0$ such that $ 0 \le cg(n) \le f(n) $ for all $ n \ge n_0 \}$

  • Theta 符號 $\Theta$
    $\Theta(g(n)) = \{f(n): $ there exist positive constants $c_1$,$c_2$, and $n_0$ such that $ 0 \le c_1g(n) \le f(n) \le c_2g(n) $ for all $ n \ge n_0 \}$

(以上定義取自 《Introduction to Algorithms》)
有三個函數圖形還蠻好理解的，只是恩，在書上…
